## Leet Code 문제 풀이


### 다시 봐야할 알고리즘 문제

- 22번 : Generate Parentheses

    - 파이썬 내장함수 product를 사용 안하고 DFS로 풀이하는 방법에 대해 다시 익힐 필요가 있음
    - 외국 개발자들은 product 함수를 안쓰고 DFS로 푸는 것 같다 (47번도 동일한 상황) 

- 47번 : Permutation2

    - 22번과 동일한 상황임 (product 안쓰고, DFS로 연산 효율성 개선)

- 20, 22, 32번 : 괄호를 사용한 스택 문제

    - 번호 뒤쪽으로 갈 수록 단순한 스택에서 인덱스나 몇 가지 기법을 추가해 풀어야하는 경우로 발전하고 있다.

- 1155번 : DFS 문제

    - 재귀와 DFS 문제를 제대로 다시 이해하고 문제를 봐야한다.

- 1160번 : Dictionary를 극한으로 잘 활용한 문제

    - 이 문제 속 풀이가 속도가 상위 98%에 달할 정도로 dictionary를 잘 활용한 문제가 아닌가 싶다.

- 인터뷰 문제 중, 조건에 맞게 코딩하는 시뮬레이션 문제 [LicenseKeyFormatting](./interview_questions/03_licenseKeyFormatting.py)는 어려운 코드가 들어가는 것은 아니지만 유의깊게 문제를 다시 볼 필요가 있다.

### 내 알고리즘 풀이 중 효율성을 개선할 필요성이 있는 방법들

- product 내장 함수
    - 데카르트 곱을 쓰는 순열과 조합 내장 함수 중 하나인데, `repeat = n` 조건에서 확인할 수 있듯, n개 만큼 반복문을 겹치는 효과를 만들어 속도가 매우 느리다.
    - `repeat=3`까지는 어느정도 인정할 수 있지만, 연산 효율성이 중요한 알고리즘 풀이에서는 인정되기 어렵다.
    - Leet Code 22, 47번에서 보듯 이걸 DFS로 개선이 가능한 것 같은데, 아직 이해를 좀 못했다.
    - 1155번을 product함수를 사용하다가 `repeat = 30` 되는 경우가 발생해 속도 개선을 도저히 못해 못풀었던 기억이 난다.
